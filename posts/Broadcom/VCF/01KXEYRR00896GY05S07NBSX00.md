# 개요

## 1. Supervisor란?

**Supervisor**는 VMware Cloud Foundation(VCF) 9에서 Kubernetes 기반의 클라우드 네이티브 서비스를 제공하기 위한 핵심 플랫폼이다. 기존 vSphere 인프라 위에 Kubernetes 제어 플레인을 통합하여, 가상머신(VM)과 컨테이너를 하나의 플랫폼에서 일관되게 관리할 수 있도록 설계되었다.

사용자는 더 이상 별도의 Kubernetes 관리 클러스터를 구축하거나 운영할 필요 없이, vCenter를 통해 Kubernetes 클러스터를 생성하고 관리할 수 있으며, 인프라 관리자와 애플리케이션 개발자가 동일한 플랫폼을 활용하면서도 각자의 역할에 맞는 리소스를 사용할 수 있다.

VCF 9에서는 기존 Tanzu Kubernetes Grid(TKG) 중심의 구조에서 한 단계 발전하여 **vSphere Kubernetes Service(VKS)**를 기본 Kubernetes 서비스로 제공하며, Supervisor는 이러한 VKS 클러스터를 생성하고 관리하는 기반이 된다.

---

## 2. Supervisor의 역할

Supervisor는 단순한 Kubernetes 클러스터가 아니라, vSphere 리소스를 Kubernetes API로 추상화하는 **관리(Control Plane)** 역할을 수행한다.

주요 역할은 다음과 같다.

- Kubernetes API 제공
- Namespace 기반 멀티테넌시
- vSphere 리소스(CPU, Memory, Storage, Network) 관리
- VKS(Kubernetes Cluster) Lifecycle 관리
- VM Service 및 Kubernetes 워크로드 관리
- CSI, CNI 등 플랫폼 서비스 제공

즉, Supervisor는 **인프라와 Kubernetes를 연결하는 브리지(Bridge)** 역할을 담당한다.

---

## 3. Supervisor 아키텍처

Supervisor는 ESXi 호스트와 vCenter 위에서 동작하며, 여러 구성 요소가 함께 Kubernetes 플랫폼을 구성한다.

대표적인 구성 요소는 다음과 같다.

- **vCenter Server**
  - Supervisor와 Kubernetes 리소스를 관리하는 중앙 관리 서버

- **Supervisor Control Plane**
  - Kubernetes API Server
  - Controller Manager
  - Scheduler
  - ETCD

- **Spherelet**
  - ESXi를 Kubernetes Node처럼 동작하도록 하는 구성 요소

- **vSphere Kubernetes Service(VKS)**
  - Supervisor를 통해 생성되는 Kubernetes Cluster

- **Cloud Provider / CSI / CNI**
  - Storage, Networking, LoadBalancer 등의 Kubernetes 기능 제공

---

## 4. Supervisor와 Workload Cluster

Supervisor 자체도 Kubernetes Cluster이지만, 일반적인 애플리케이션을 실행하기 위한 클러스터는 아니다.

실제 사용자 애플리케이션은 Supervisor가 생성하는 **VKS Cluster(Workload Cluster)** 위에서 실행된다.

## 5. vSphere Namespace

Supervisor에서는 Kubernetes Namespace가 단순한 논리적 구분을 넘어 **리소스 할당 단위**로 사용된다.

Namespace에는 다음과 같은 리소스를 연결할 수 있다.

- Compute(Resource Pool)
- Storage Policy
- Network
- Permissions(RBAC)
- VM Classes
- Content Library

이를 통해 하나의 Supervisor에서 여러 조직이나 프로젝트가 독립적으로 Kubernetes Cluster를 운영할 수 있다.

---

## 6. VCF 9에서 Supervisor의 변화

VCF 9에서는 Kubernetes 플랫폼이 이전 버전보다 더욱 단순하고 표준 Kubernetes에 가까운 형태로 발전하였다.

대표적인 변화는 다음과 같다.

- VKS(vSphere Kubernetes Service) 중심의 Kubernetes 제공
- Cluster API(CAPI) 기반 Cluster Lifecycle 관리
- Kubernetes 버전 관리 단순화
- VM Service와의 통합 강화
- GitOps 및 자동화 환경과의 연계성 향상

이를 통해 Kubernetes 운영 방식이 더욱 표준화되었으며, 기존 Kubernetes 생태계의 다양한 도구와도 자연스럽게 연동할 수 있게 되었다.

---

## 정리

Supervisor는 단순히 Kubernetes를 실행하기 위한 클러스터가 아니라, **vSphere와 Kubernetes를 통합하는 관리 플랫폼**이다.

VCF 9에서는 Supervisor를 중심으로 VKS Cluster를 생성하고 관리하며, 사용자는 표준 Kubernetes 환경을 활용하면서도 vSphere의 컴퓨팅, 스토리지, 네트워크 기능을 그대로 사용할 수 있다.

결과적으로 Supervisor는 인프라 관리자에게는 일관된 리소스 관리 환경을, 개발자에게는 표준 Kubernetes 기반의 애플리케이션 실행 환경을 제공하는 핵심 구성 요소라고 할 수 있다.
